Laravel на Docker контейнерах

Как запустить Laravel приложение на Docker контейнерах

Чтобы установить фреймворк Laravel, необходимо иметь, как минимум следующие сервисы:

  • Веб сервер, в нашем случае это Nginx
  • php-fpm для обрботки скриптов PHP
  • базу данных MySQL
  • Node.js для работы с фронтендом

Каждый сервис будет вполняться в отдельном контейнере и в случае если понадобится еще какой-нибудь (например Redis) мы его так же запустим в отдельном контейнере. Управлять и конфигурировать эти контейнеры будет Docker Compose. Чтобы не писать каждый раз все с нуля, я подготовил стандартный набор контейнеров для Laravel приложения, положил его в репозиторий.

Клонируйте данный репозиторий в папку ~/my-projects/project-1/laravel-docker

cd ~/my-projects/project-1
git clone https://github.com/acwstudio/laravel-docker laravel-docker-1
cd laravel-docker-1

Внутри папки ~/my-projects/project-1/laravel-docker-1 будет следующая файловая структура

Внутри папки ~/my-projects/project-1/laravel-docker-1 нам интересны следующие папки и файлы

1. data/db/mysql/ Volume с базой данных из контейнера MySQL
2. data/redis/ Volume с базой данных из контейнера Redis
3. .env.example Основа для создания своего .env файла
4. etc/db/mysql/ Volume c конфигом для контейнера MySQL
5. etc/nginx/conf.d/ Volume c конфигом для контейнера Nginx
6. etc/php/php.ini Volume c конфигом для контейнера PHP
7. etc/php/Dockerfile Создание image для контейнера PHP
8. etc/redis/ Конфиг для Redis
9. etc/redis/Dockerfile Создание image для контейнера Redis
10. example-files/docker-compose.yml.example Основа для docker-compose.yml файла
11. example-files/laravel.conf.example Основа для Nginx конфига
12. example-files/my.cnf.example Основа для MySQL конфига
13. example-files/php.ini.example Основа для PHP конфига
14. example-files/redis.conf.example Основа для Redis конфига
15. laravel-app/ Volume c кодом Laravel приложения для контейнеров PHP и Nginx

Произведем необходимые настройки

  1. В папке laravel-app/ создадим симлинк project на код Laravel приложения.
    cd ~/my-projects/project-1/laravel-docker-1
    ln -s ~/my-projects/project-1/laravel-app-1 project
  2. Сделаем копию файла .env.example под именем .env и откроем для редактирования
    cp .env.example .env
    nano .env
    Оставляем только указанные ниже строки, остальное комментим или удаляем
    # App settings
    APP_NAME=LarApp1
    APP_PATH=./laravel-app/project
    # Port Mappings
    PORT_DATABASE=3300
    PORT_HTTP=8000
    PORT_HTTPS=8100
    # mysql settings
    MYSQL_DATABASE=db_1
    MYSQL_ROOT_USER=root
    MYSQL_ROOT_PASSWORD=root
    MYSQL_USER=dev
    MYSQL_PASSWORD=dev
